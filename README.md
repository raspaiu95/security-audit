# 🔐 Security Audit Automation

## 1. 개요 (Overview)
본 프로젝트는 2026 주요정보통신기반시설 기술적 취약점 분석·평가방법 상세가이드를 기준으로  
Linux 서버 환경(Ubuntu, Rocky Linux)을 대상으로 보안 취약점 점검을 자동 수행하고,  
진단 결과를 표준화된 형태로 정리하기 위해 개발한 보안 점검 자동화 도구입니다.

운영 환경에서 반복적으로 수행되던 수동 점검 작업을 자동화하여  
진단 소요 시간을 단축하고 결과의 정합성과 재사용성을 높이는 것을 목표로 합니다.

---

## 2. 개발 배경 (Background)
다수의 서버를 운영하는 환경에서 다음과 같은 문제를 경험했습니다.

- OS별로 상이한 점검 항목 및 스크립트 관리 부담
- 진단 결과가 텍스트 파일 형태로 산출되어 후처리(취합/요약)의 어려움
- 동일 점검 항목이 반복 출력되는 결과 파일로 인한 분석 오류
- 인프라 환경 변화에 따라 스크립트를 지속적으로 수정해야 하는 문제

이를 해결하기 위해 OS별 점검 스크립트와  
결과 파싱/정규화를 분리한 구조로 도구를 설계했습니다.

## ✨ 5. 주요 기능 (Key Features)

- Linux OS별 보안 취약점 자동 점검
  - Ubuntu
  - Rocky Linux
- 점검 결과 텍스트 파일 자동 생성
- 진단 결과 파싱 및 표준 포맷(CSV / Excel) 변환
- 중복 점검 항목 병합 및 결과 정규화
- 파일명 기반 호스트 정보(IP / Hostname / OS) 자동 추출
- 항목별 취약 / 양호 / 수동확인 통계 산출
- 서버(IP) 단위 보안 준수율(%) 자동 계산

```text

├── statistics.py          # 진단 결과 파싱 및 통계/준수율 산출
```

## ▶️ 7. 사용 방법 (Usage)

### 1) 대상 서버에서 보안 점검 수행

bash ubuntu_audit.sh  
또는  
bash rocky_audit.sh

### 2) 로컬 환경에서 결과 분석 및 통계 생성

python statistics.py

## 📊 8. 결과물 (Output)
